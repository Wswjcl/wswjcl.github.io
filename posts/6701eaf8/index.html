RAG 系列（三）：嵌入模型 — 让文本可计算

发表于2026-07-09|更新于2026-07-10|技术
|浏览量:

文本分块之后，RAG 管线的下一步是将文本转换为向量——这就是嵌入模型（Embedding Model）的职责。嵌入模型将人类可读的文本映射到高维向量空间中，使得语义相近的文本在向量空间中距离更近，从而让”找相似内容”这一操作变为数学上的向量近邻搜索。

本文将深入探讨嵌入模型的原理、训练方法、主流模型选型与实战考量。

一、从词嵌入到文档嵌入

1.1 词嵌入的演进

自然语言的离散性是机器处理的核心障碍。嵌入（Embedding）的核心思想是将离散符号映射为连续向量，使得语义关系可以通过向量运算来表达。

阶段 代表模型 特点
One-Hot — 高维稀疏，无语义，词间正交
静态词嵌入 Word2Vec, GloVe, FastText 低维稠密，词间有语义关系，但一词一向量
上下文化嵌入 BERT, GPT 同一词在不同上下文有不同表示，但需额外池化
文本嵌入 BGE, E5, GTE, Cohere 直接输出文本级的语义向量，专为检索设计

1.2 从词向量到文档向量

RAG 需要的不是单个词的向量，而是整段文本（查询或文档块）的向量表示。从词向量到文档向量，常见方法有：

  • 平均池化：将所有 token 的向量取平均。简单但信息损失大。
  • CLS token：取 BERT 类模型的 [CLS] token 输出。训练时需对 CLS 做专门优化。
  • 注意力池化：用可学习的注意力权重对 token 向量加权求和，更灵活。
  • 专用嵌入模型：直接训练一个”输入文本 → 输出向量”的模型，端到端优化。

现代 RAG 几乎全部采用专用嵌入模型，而非在 LLM 之上做池化。

二、嵌入模型的核心原理

2.1 语义空间

嵌入模型将文本映射到一个 $d$ 维向量空间（通常 $d = 768$ 或 $1024$），这个空间被称为语义空间。在语义空间中：

  • 语义相似的文本 → 向量距离近
  • 语义不同的文本 → 向量距离远
  • 语义关系可以用向量方向表示

2.2 相似度度量

在语义空间中度量两个向量的相似程度，常用三种方式：

余弦相似度（Cosine Similarity）

$$\text{sim}(\mathbf{a}, \mathbf{b}) = \frac{\mathbf{a} \cdot \mathbf{b}}{|\mathbf{a}| \cdot |\mathbf{b}|}$$

值域 $[-1, 1]$，衡量向量方向的相似性，不受向量长度影响。RAG 中最常用的度量方式。

点积（Dot Product / Inner Product）

$$\text{sim}(\mathbf{a}, \mathbf{b}) = \mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{d} a_i b_i$$

当向量已归一化（$|\mathbf{a}| = |\mathbf{b}| = 1$）时，点积等于余弦相似度。计算速度更快。

欧氏距离（Euclidean Distance）

$$\text{dist}(\mathbf{a}, \mathbf{b}) = \sqrt{\sum_{i=1}^{d} (a_i - b_i)^2}$$

值域 $[0, +\infty)$，衡量向量间的绝对距离。当向量归一化后，欧氏距离与余弦相似度单调相关。

选择建议：RAG 中推荐使用余弦相似度，因为它对向量尺度不敏感，更鲁棒。如果向量已归一化，可以直接用点积加速计算。

2.3 对比学习：嵌入模型的训练范式

现代嵌入模型几乎都基于对比学习（Contrastive Learning）进行训练。其核心思想是：拉近正样本对的距离，推远负样本对的距离。

InfoNCE 损失

最常用的对比学习损失函数：

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(q, d^+) / \tau)}{\sum_{j=1}^{N} \exp(\text{sim}(q, d_j) / \tau)}$$

其中：

  • $q$：查询（Query）的嵌入向量
  • $d^+$：正样本（与查询相关的文档）的嵌入向量
  • $d_j$：所有候选文档（包含正样本和负样本）的嵌入向量
  • $\tau$：温度超参数，控制分布的锐度
  • $N$：批次中的样本总数

直观理解：在一批样本中，让查询与正确文档的相似度尽可能高，同时压低与错误文档的相似度。

负样本策略

负样本的质量对对比学习至关重要：

策略 描述 特点
随机负采样 从语料库中随机选取 简单但区分度低（太容易区分）
批内负采样 同一 batch 内的其他样本作为负样本 高效，但负样本可能不够”难”
硬负挖掘 选取与查询相似但无关的文档作为负样本 训练信号更强，效果最好
LM-based 挖掘 用 LLM 生成困难负样本 自动化程度高

硬负样本（Hard Negatives）是提升嵌入质量的关键。例如，对于查询”什么是注意力机制”，一个好的硬负样本是”注意力缺陷多动障碍（ADHD）的诊断标准”——包含关键词但语义不同。

四、嵌入维度与语义表达能力

4.1 维度的意义

嵌入维度 $d$ 决定了语义空间的表达能力：

  • 维度太低 → 语义空间不足以区分细粒度差异（欠拟合）
  • 维度太高 → 向量稀疏，计算和存储成本增加（过拟合风险）

主流嵌入模型的维度在 768-3072 之间。经验上，768 维已经足够覆盖大多数语义信息。

4.2 维度截断（Matryoshka Embeddings）

OpenAI 的 text-embedding-3 支持维度截断：可以从 3072 维向量中截取前 256/512/1024 维使用，而不会严重损失语义信息。

原理：训练时使用 Matryoshka Representation Learning（MRL），使得向量的前 $k$ 维就是 $k$ 维空间中的最优表示。

$$\mathcal{L}{\text{MRL}} = \sum{k \in {256, 512, 1024, 3072}} \mathcal{L}{\text{InfoNCE}}(\mathbf{h}{1:k})$$

这意味着你可以用一个模型同时满足不同场景：

  • 粗筛场景：用 256 维，速度快
  • 精排场景：用 1024 维，精度高
  • 存储优化：用 512 维，节省空间

五、多语言与跨语言嵌入

5.1 多语言嵌入的挑战

不同语言的语义空间需要对齐——“cat”和”猫”在向量空间中应该距离很近。这需要：

  1. 多语言训练数据：覆盖足够多的语言对
  2. 跨语言对比学习：正样本对包含不同语言的等价文本
  3. 共享语义空间：所有语言映射到同一向量空间

5.2 跨语言检索

跨语言嵌入允许用一种语言的查询检索另一种语言的文档：

这在多语言知识库中非常有用，避免了对每种语言分别建索引的麻烦。

推荐模型：BGE-M3（支持 100+ 语言）、Cohere embed-v3（多语言优化）。

六、嵌入模型的实战考量

6.1 归一化

大多数嵌入模型输出的向量已经过归一化（$|\mathbf{h}| = 1$），但部分模型不是。建议在使用前统一做 L2 归一化：

$$\hat{\mathbf{h}} = \frac{\mathbf{h}}{|\mathbf{h}|}$$

归一化后余弦相似度 = 点积，可以简化计算。

6.2 查询与文档的嵌入差异

许多嵌入模型要求查询和文档使用不同的前缀（如 E5 的 query: vs passage:）。这是因为训练时查询和文档的语义分布不同，模型需要知道输入的角色。

常见错误：对查询和文档使用相同的前缀，或不使用前缀。这会导致检索效果显著下降。

6.3 嵌入缓存与增量更新

  • 嵌入缓存：对不变的分块缓存其嵌入向量，避免重复计算
  • 增量更新：新增文档时只计算新分块的嵌入，不影响已有索引
  • 模型切换：更换嵌入模型时需要重新计算所有嵌入，成本较高

6.4 批量推理优化

对大量文档做嵌入时，应使用批量推理（Batch Inference）而非逐条处理：

批量推理可以充分利用 GPU 并行能力，速度提升 10-50 倍。

七、嵌入模型的评估

7.1 MTEB 排行榜

MTEB（Massive Text Embedding Benchmark） 是目前最权威的嵌入模型评估基准，涵盖 7 个任务类别：

任务类别 说明
Classification 文本分类
Clustering 文本聚类
Pair Classification 文本对分类
Reranking 重排序
Retrieval 信息检索（RAG 最相关）
Semantic Textual Similarity 语义相似度
Summarization 摘要质量

7.2 自建评估

MTEB 是通用评估，你的具体场景可能不同。建议自建评估：

  1. 构建一批（查询, 相关文档, 无关文档）三元组
  2. 计算查询与所有文档的相似度
  3. 检查相关文档是否排在 Top-K 中
  4. 计算 Recall@K、MRR、nDCG 等指标

7.3 关键指标

指标 含义 适用场景
Recall@K 前 K 个结果中包含正确答案的比例 RAG 检索评估
MRR 第一个正确答案的排名倒数 单答案检索
nDCG@K 考虑排序位置的归一化增益 多级相关性评估

八、小结

嵌入模型是 RAG 的”感知器官”，将文本转换为可计算的向量，是检索质量的基础。本文的核心要点：

  • 嵌入模型从词嵌入演进到文档嵌入，现代 RAG 使用专用的文本嵌入模型
  • 对比学习是训练范式的核心，InfoNCE 损失 + 硬负样本是关键
  • 选型需考虑语言、部署方式、上下文长度，中文推荐 BGE / GTE 系列
  • 维度截断（Matryoshka）技术允许灵活权衡精度与效率
  • 使用时注意指令前缀、归一化、批量推理等实战细节

一、什么是 RAG？RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术范式。其核心思想是：在大型语言模型（LLM）生成回答之前，先从外部知识库中检索相关信息，将检索到的内容作为上下文提供给 LLM，从而让模型的回答基于事实而非纯粹的参数记忆。 2020 年，Lewis 等人在论文 Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks 中首次系统性地提出了 RAG 框架，将预训练的检索器与序列到序列模型相结合，在开放域问答等任务上取得了显著提升。 123456flowchart LR Q["用户查询"] --> R["检索器<br/>Retriever"] R --> C["检索到的<br/>相关文档"] C --> G["生成器<br/>Generator (LLM)"] Q...
