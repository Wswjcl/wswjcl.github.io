Transformer 详解

发表于2026-07-01|更新于2026-07-05|技术
|浏览量:

一、背景与动机

在 Transformer 出现之前，序列建模任务（机器翻译、文本生成等）的主流架构是 RNN 及其变体 LSTM / GRU。这些模型虽然在当时取得了不错的效果，但存在几个根本性的局限：

  1. 顺序计算瓶颈：RNN 必须逐步处理序列中的每个时间步，无法充分并行化。序列越长，训练越慢。
  2. 长距离依赖问题：尽管 LSTM 引入了门控机制来缓解梯度消失，但在实际中捕获超长距离的依赖关系仍然困难。
  3. 信息压缩瓶颈：RNN 将整个序列的信息压缩到一个固定维度的隐状态中，信息容易丢失。

2017 年，Vaswani 等人在论文 Attention Is All You Need 中提出了 Transformer 架构，完全摒弃了循环和卷积结构，仅依靠注意力机制（Attention）来建模序列间的依赖关系。这一设计带来了两大优势：

  • 全并行计算：所有位置可以同时计算，极大提升了训练效率。
  • 全局感受野：每个位置可以直接关注序列中任意其他位置，天然支持长距离依赖。

Transformer 的出现彻底改变了 NLP 乃至整个深度学习的格局，是当今大语言模型（GPT、BERT、LLaMA 等）的基石。

二、整体架构

Transformer 采用经典的 Encoder-Decoder 结构：

  • Encoder：由 N 个相同的 Encoder Layer 堆叠而成（原论文 N=6）。每一层包含两个子层：Multi-Head Self-Attention + Feed-Forward Network，每个子层后接残差连接与 Layer Normalization。
  • Decoder：同样由 N 个相同的 Decoder Layer 堆叠而成。每一层包含三个子层：Masked Multi-Head Self-Attention + Multi-Head Cross-Attention + Feed-Forward Network，同样每个子层后接残差连接与 Layer Normalization。

输入在进入 Encoder/Decoder 之前，会先经过 词嵌入（Embedding） 和 位置编码（Positional Encoding） 的处理。

三、Self-Attention 自注意力机制

Self-Attention 是 Transformer 的核心创新。其基本思想是：对于序列中的每个位置，通过计算它与序列中所有其他位置的关联程度（注意力权重），来加权聚合信息，从而得到该位置的上下文表示。

3.1 Q、K、V 的含义

Self-Attention 借鉴了信息检索的思想，引入三个矩阵将输入映射为：

  • Query（Q）：查询向量，表示”我在找什么”
  • Key（K）：键向量，表示”我有什么特征”
  • Value（V）：值向量，表示”我的实际内容”

计算方式：

$$Q = XW^Q, \quad K = XW^K, \quad V = XW^V$$

其中 $X$ 是输入矩阵，$W^Q, W^K, W^V$ 是可学习的参数矩阵。

3.2 注意力计算过程

Scaled Dot-Product Attention 的计算公式为：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

逐步拆解：

  1. 计算注意力分数：$QK^T$，得到一个 $n \times n$ 的矩阵，其中第 $(i, j)$ 个元素表示位置 $i$ 对位置 $j$ 的原始关注程度。
  2. 缩放：除以 $\sqrt{d_k}$。这是因为当 $d_k$ 较大时，点积的方差会增大，导致 softmax 进入梯度极小的饱和区。缩放因子使方差稳定在 1 附近。
  3. Softmax 归一化：对每一行做 softmax，将注意力分数转化为概率分布（和为 1，非负）。
  4. 加权求和：用注意力权重对 V 加权聚合，得到每个位置的上下文表示。

3.3 直觉理解

可以把 Self-Attention 想象成一个”智能的信息聚合器”：每个词都会向序列中所有其他词”提问”，根据问题的匹配程度决定从对方那里获取多少信息。比如在句子”这只猫很可爱，它喜欢吃鱼”中，”它”这个词的 Query 会和”猫”的 Key 高度匹配，从而在”它”的表示中融入”猫”的信息。

四、Multi-Head Attention 多头注意力

4.1 为什么需要多头？

单头注意力只能学习一种关注模式，但语言中的依赖关系是多样的——语法关系、语义关系、指代关系等往往需要不同的关注模式。Multi-Head Attention 允许模型同时从不同的表示子空间中关注不同位置的信息。

4.2 计算方式

将 Q、K、V 分别投影到 h 个不同的子空间（原论文 h=8），在每个子空间独立计算 Scaled Dot-Product Attention，然后将结果拼接并做一次线性投影：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h)W^O$$

其中：

$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

每个头的维度为 $d_k = d_{\text{model}} / h$，这样多头计算的总代价与单头全维度注意力相当。

4.3 多头的意义

  • 多样性：不同头可以关注不同类型的依赖关系（如一个头关注相邻词，另一个头关注远距离指代）。
  • 鲁棒性：多个头的集成使模型对单一关注模式的偏差更加鲁棒。
  • 可解释性：可视化不同头的注意力分布，可以揭示模型学到的不同关注模式。

五、Positional Encoding 位置编码

5.1 为什么需要位置编码？

Self-Attention 本身是置换不变的（permutation invariant）——打乱输入顺序，输出只是相应地打乱，但数值关系不变。这意味着模型本身无法感知序列中元素的顺序信息，而顺序对语言至关重要。

RNN 通过逐步处理的顺序天然编码了位置信息，Transformer 则需要显式注入位置信息。

5.2 正弦/余弦位置编码

原论文使用不同频率的正弦和余弦函数：

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

其中 $pos$ 是位置索引，$i$ 是维度索引。

5.3 设计直觉

  • 不同频率：低维度使用高频信号（快速变化），高维度使用低频信号（缓慢变化），类似于二进制编码的不同位，使每个位置都有独特的编码。
  • 相对位置关系：对于任意固定偏移 $k$，$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性变换，这使得模型能够学习相对位置关系。
  • 泛化性：正弦/余弦函数的值域有界，可以对训练中未见过的序列长度进行合理外推。

5.4 其他位置编码方案

  • 可学习位置编码：将位置编码作为可训练参数（BERT 采用此方案），灵活性更高但无法外推到更长序列。
  • 相对位置编码：在注意力计算中直接编码相对距离（如 Transformer-XL、T5）。
  • RoPE（旋转位置编码）：通过旋转矩阵编码相对位置，被 RoPE、LLaMA 等模型采用，具有优良的外推性质。

六、Feed-Forward Network 前馈网络

每个 Encoder/Decoder Layer 中的 Feed-Forward Network（FFN）是一个逐位置（position-wise）的两层全连接网络：

$$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$

即先线性映射到更高维度（原论文从 512 扩展到 2048），经过 ReLU 激活，再映射回原始维度。

作用

  • 非线性变换：Attention 本质上是加权求和（线性操作），FFN 引入非线性，增强模型的表达能力。
  • 特征变换：可以理解为对每个位置的表示进行独立的”特征加工”，类似于对每个词做一次 MLP。
  • 记忆功能：有研究表明 FFN 类似于键值记忆网络，$W_1$ 的行向量充当”键”，$W_2$ 的列向量充当”值”。

现代变体中，激活函数常替换为 GELU（GPT）或 SwiGLU（LLaMA），并在 FFN 中引入门控机制。

七、Layer Normalization 与残差连接

7.1 残差连接（Residual Connection）

每个子层的输出都加上该子层的输入：

$$\text{output} = \text{LayerNorm}(x + \text{Sublayer}(x))$$

残差连接的作用：

  • 梯度流通：为梯度提供一条直接回传的”高速公路”，缓解深层网络的梯度消失问题。
  • 信息保留：确保子层学到的至少是”增量”信息，即使子层输出为零，原始信息也不会丢失。

7.2 Layer Normalization

对每个样本的特征维度做归一化：

$$\text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sigma} + \beta$$

其中 $\mu$ 和 $\sigma$ 是该样本所有特征的均值和标准差，$\gamma$ 和 $\beta$ 是可学习的缩放和偏移参数。

Layer Norm 的作用：

  • 稳定训练：控制激活值的数值范围，防止梯度爆炸/消失。
  • 加速收敛：使优化地形更加平滑，允许使用更大的学习率。

7.3 Pre-Norm vs Post-Norm

原论文使用 Post-Norm（先残差连接，再做 LayerNorm），但后来的实践发现 Pre-Norm（先做 LayerNorm，再进入子层）训练更稳定：

Pre-Norm 已成为主流选择，GPT-2 及后续的大模型普遍采用 Pre-Norm。

八、Mask 机制

8.1 Padding Mask

输入序列通常需要 padding 到统一长度，但 padding 位置不应参与注意力计算。Padding Mask 将 padding 位置的注意力分数设为 $-\infty$，经过 softmax 后权重趋近于 0。

8.2 Causal Mask（序列掩码）

在 Decoder 的 Self-Attention 中，位置 $i$ 不能关注位置 $j > i$ 的信息（即”未来”信息），否则就变成了”偷看答案”。Causal Mask 通过将上三角位置的注意力分数设为 $-\infty$ 来实现：

这确保了 Decoder 的自回归性质：生成第 $i$ 个词时，只能看到第 $1$ 到第 $i-1$ 个词。

九、Decoder 解码器

9.1 结构

每个 Decoder Layer 包含三个子层：

  1. Masked Multi-Head Self-Attention：对已生成的输出序列做自注意力，加上 Causal Mask 防止看到未来信息。
  2. Multi-Head Cross-Attention：Query 来自 Decoder，Key 和 Value 来自 Encoder 的输出。这是 Decoder 获取源序列信息的关键通道。
  3. Feed-Forward Network：与前述相同。

9.2 自回归生成

Decoder 采用自回归（autoregressive）方式逐步生成输出：

  1. 起始时输入 <s>（开始标记），预测第一个输出词。
  2. 将预测结果拼接到输入中，预测第二个词。
  3. 重复直到生成 <eos>（结束标记）或达到最大长度。

每一步的预测都基于之前所有已生成的词，因此生成过程是串行的，这也是 Transformer 推理速度的主要瓶颈之一。

十、训练与推理

10.1 训练

  • Teacher Forcing：训练时使用真实的目标序列作为 Decoder 输入，而非模型自己的预测。这避免了误差累积，加速收敛。
  • Label Smoothing：原论文使用了 $\epsilon = 0.1$ 的标签平滑，防止模型过度自信，提升泛化能力。
  • 损失函数：交叉熵损失，对每个位置的预测计算损失并求平均。

10.2 推理

  • 贪心解码：每步选择概率最高的词，简单但可能错过全局最优序列。
  • Beam Search：维护 beam_size 个候选序列，每步扩展所有候选并保留得分最高的 beam_size 个。beam_size 越大搜索越充分，但速度越慢。
  • 采样解码：从预测分布中随机采样，引入温度参数控制随机性。Top-k 和 Top-p（nucleus）采样是常用的截断策略。

十一、变体与演进

Transformer 的 Encoder-Decoder 架构衍生出三大流派：

11.1 Encoder-Only：BERT

  • 仅保留 Encoder 部分，使用双向（无 Mask）Self-Attention。
  • 通过 Masked Language Model（MLM）和 Next Sentence Prediction（NSP）进行预训练。
  • 擅长理解型任务：文本分类、命名实体识别、阅读理解等。
  • 代表模型：BERT、RoBERTa、ALBERT、DeBERTa。

11.2 Decoder-Only：GPT

  • 仅保留 Decoder 部分，使用 Masked Self-Attention（因果注意力）。
  • 通过自回归语言建模进行预训练（预测下一个词）。
  • 擅长生成型任务：文本生成、对话、代码生成等。
  • 代表模型：GPT 系列、LLaMA、Mistral、Qwen。
  • 当前大模型的主流架构，因其在生成任务上的优异表现和 Scaling Law 的加持而成为事实标准。

11.3 Encoder-Decoder：T5 / BART

  • 保留完整的 Encoder-Decoder 结构。
  • T5 将所有 NLP 任务统一为”文本到文本”的格式。
  • BART 使用去噪自编码器进行预训练。
  • 擅长序列到序列任务：机器翻译、文本摘要等。
  • 代表模型：T5、BART、mBART、Flan-T5。

11.4 架构改进趋势

改进方向 具体方法 代表模型
位置编码 RoPE、ALiBi LLaMA、BLOOM
归一化 Pre-Norm、RMSNorm GPT-2、LLaMA
激活函数 GELU、SwiGLU GPT、LLaMA
注意力优化 Flash Attention、MQA、GQA GPT-4、LLaMA-2
稀疏注意力 Sliding Window Attention Mistral
长上下文 YaRN、NTK-aware 插值 各类长上下文模型

十二、总结

Transformer 的核心贡献可以概括为：

  1. 注意力即全部：证明了纯注意力机制足以建模序列依赖，无需循环或卷积。
  2. 并行化：彻底释放了序列建模的计算潜力，使得大规模训练成为可能。
  3. 可扩展性：Transformer 架构展现出优秀的 Scaling Law——模型越大、数据越多、算力越强，性能越好。这一特性直接催生了 GPT-4、LLaMA 等大语言模型。
  4. 通用性：从 NLP 扩展到 CV（ViT）、语音（Whisper）、多模态（GPT-4V）等领域，成为深度学习的”通用架构”。

从 2017 年的一篇机器翻译论文，到如今支撑整个大模型时代的基石，Transformer 的影响力远超其最初的设计目标。理解 Transformer，是理解当代 AI 的必经之路。
